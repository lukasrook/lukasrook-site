Kill the 404s: Hosting a Project Site on GitHub Pages

Today’s lesson: GitHub Pages project sites live under /repo-name/. If your links assume the root domain, you’ll ship 404s—especially on mobile.

The bug

I launched a home page at lukasrook.github.io/lukasrook-site/ and added a blog overview under /blog/. On production every link pointed to https://lukasrook.github.io/blog/ which doesn’t exist.

The fix

  1. Confirmed the real URLs: https://lukasrook.github.io/lukasrook-site/blog/ etc.
  2. Replaced every hard-coded /blog/… reference with /lukasrook-site/blog/….
  3. Committed & pushed so GitHub Pages rebuilt; verified with curl -I.
  4. Hard-refreshed mobile Safari to clear the cached 404.

Takeaway

If you deploy a project site on GitHub Pages, either reference assets with the baseurl variable or manually prefix every internal link with the repo slug. Skip that step and visitors will bounce from a dead page.
